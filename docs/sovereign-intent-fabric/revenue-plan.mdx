# Revenue Plan

The Sovereign Intent Fabric follows a disciplined 36-month revenue plan. Enterprise-first. No VC dependency. No token speculation. Revenue is tied to measurable risk reduction, cost savings, and execution quality — not attention, engagement, or data harvesting.

---

## Unit Economics

The base monetizable product is the **Enterprise Sovereign AI Node** — software runtime + orchestration + security stack, deployed on-premises or on customer-controlled infrastructure.

| Metric | Value |
|---|---|
| **Annual revenue per enterprise** | $48,000 ($4,000/month) |
| **Target customer** | Mid-sized enterprise (200-500 employees) |
| **Customer Acquisition Cost (CAC)** | $12,000 (amortized over 3-year retention = $4,000/year) |
| **Annual support & ops** | $10,000/year per customer |
| **Infra overhead** | $2,000/year per customer |
| **R&D allocation** | $9,600/year per customer (20% of revenue) |
| **Net contribution per customer** | $22,400/year |
| **Gross margin** | ~46% |

### Cost Breakdown Per Enterprise (Annual)

|---|---|

## 36-Month Revenue Projections

### Phase 1: Survive & Prove (Year 0-1)

| Metric | Value |
|---|---|
| **Customers** | 3-5 |
| **Revenue** | ~$250,000 |
| **Net contribution** | ~$110,000 |
| **Team size** | Founder-led, no hires |
| **Focus** | One vertical (law firms), one use case, measurable delta |

**Deliverables shipped**: Sovereign Identity (enterprise-bound, TPM-backed), Edge AI Runtime (confidential inference), Scoped Agent Execution, audit logs, deployment playbook.

**Sales pitch**: "Run AI locally. Reduce risk. Reduce cost." Not "rearchitect civilization."

**Success criteria**: Uptime, security posture, reproducible deployment, incident response discipline, ability to survive audits.

### Phase 2: Stabilize & Standardize (Year 1-2)

| Metric | Value |
|---|---|
| **Customers** | 10-20 |
| **Revenue** | ~$1,000,000 |
| **Net contribution** | ~$430,000 |
| **Team size** | Small team (first security lead, support engineer) |
| **Focus** | Extract primitives, internal spec v0, SOC2 compliance |

**What happens**: Every deployment reveals repeating patterns — identity schema, agent contract format, execution descriptor, attestation structure. These become the internal protocol specification v0.

**Key milestones**:
- First security lead hired
- Third-party security audit completed
- SOC2 minimum compliance achieved
- Internal protocol spec v0 documented

### Phase 3: Embedded Standard (Year 2-3)

| Metric | Value |
|---|---|
| **Customers** | 40-60 |
| **Revenue** | ~$3,000,000 |
| **Net contribution** | ~$1,300,000 |
| **Team size** | Dedicated support, sales, engineering |
| **Focus** | SDK release, vendor integration, proto-protocol |

**What happens**: Other vendors want integration. Instead of custom APIs, the standard execution contract format becomes the integration point. SDK opens. Not governance, not token, not manifesto — just interoperability.

**Key milestones**:
- SDK published for execution contract format
- First third-party vendor integrations
- Revenue sufficient to fund protocol development
- Case studies from 3+ verticals

### Phase 4: Controlled Expansion (Year 3-4)

## Customer Cost Comparison

Using a mid-sized financial services firm (250 employees, 120 active AI users, 180,000 queries/month) as the reference case:

### Cloud AI Model (Current State)

| Cost Component | Annual Cost |
|---|---|
| Raw API cost (180K queries x $0.015) | $32,400 |
| Data egress + compliance + security review + risk premium | ~$57,600 |
| **Total AI-related cost** | **~$90,000** |

### Sovereign Node Model (SIF)

| Cost Component | Annual Cost |
|---|---|
| Hardware amortization (2x GPU workstation, 3-year lifecycle) | $4,333 |
| Software subscription | $48,000 |
| Operations + electricity | $12,000 |
| **Total cost** | **~$64,333** |

**Annual savings**: ~$25,667 (28.5% reduction) — before accounting for data sovereignty value, compliance risk reduction, vendor lock-in avoidance, and latency improvement.

## What the Revenue Plan is NOT

- Not VC-dependent. No fundraising milestones. Revenue funds the next phase.
- Not token-based. No speculation loops. Usage-based compensation tied to verified workload.
- Not ad-driven. Revenue must never depend on engagement, data harvesting, or attention metrics.
- Not hardware-first. Software runtime + hardened deployment layer first. Hardware certification later.

## Kill Criteria

| Signal | Action |
|---|---|
| Zero customers after 12 months | Pivot vertical, not vision |
| Customer churn exceeds 30% annually | Product-market fit failure — investigate |
| Margin drops below 30% | Over-engineering or underpricing — restructure |
| Sales cycle exceeds 12 months consistently | Wrong vertical — find faster decision cycles |
| Revenue requires ad-based supplements | Architecture integrity failure — stop |
